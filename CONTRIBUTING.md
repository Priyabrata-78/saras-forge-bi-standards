## Workflows to document

1. **Submit a new prompt** — test it, tag it (tool + use case + known failure mode), post to #forge-bi-standards, Maintainer reviews and merges.
2. **Submit a new limitation** — describe the constraint, workaround, and which accounts/datasets it affects.
3. **Propose a standard change** — open a ClickUp task for Forge Maintainer review before editing `universal/CROSS_TOOL_STANDARDS.md`.
